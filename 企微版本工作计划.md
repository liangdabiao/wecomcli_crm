# 企业微信 CLI 版 — 10 大工作流 Skill 实施计划

> 参照飞书版 10 大工作流 Skill，基于 wecom-cli (企业微信 CLI) 重新实现。
> 数据底座：企业微信智能表格 (smartsheet) + 文档 (doc) + 消息 (msg)

---

## 一、飞书 vs 企业微信能力映射

| 飞书模块 | 企业微信 CLI 对应 | 状态 | 说明 |
|---------|-----------------|------|------|
| lark-im (消息) | `wecom-cli msg` | ✅ 完全支持 | 单聊/群聊收发、媒体下载 |
| lark-calendar (日历) | `wecom-cli schedule` | ✅ 完全支持 | 日程 CRUD、闲忙查询 |
| lark-task (任务) | `wecom-cli todo` | ✅ 完全支持 | 待办创建/查询/更新/删除 |
| lark-vc (会议) | `wecom-cli meeting` | ✅ 完全支持 | 会议创建/查询/管理 |
| lark-doc (云文档) | `wecom-cli doc` | ✅ 完全支持 | 文档创建/读取/编辑 |
| lark-base (多维表格) | `wecom-cli doc smartsheet` | ✅ 完全支持 | 子表/字段/记录完整 CRUD |
| lark-contact (通讯录) | `wecom-cli contact` | ✅ 完全支持 | 成员查询/搜索 |
| lark-mail (邮件) | — | ❌ 无 | CRM 和晨间简报中用消息/日程替代 |
| lark-minutes (妙记) | — | ❌ 无 | 会议待办用会议详情+手动输入替代 |
| lark-wiki (知识库) | — | ❌ 无 | 用智能表格索引+文档存储替代 |
| lark-sheets (电子表格) | — | ❌ 无 | 用智能表格记录+文档报告替代 |
| lark-drive (云空间) | — | ❌ 无 | 用文档存储替代 |

**核心差异总结：**
- 企业微信 CLI **不具备**邮件、妙记、知识库、电子表格、云空间模块
- 所有数据存储统一使用 **智能表格 (smartsheet)** — 与现有 CRM 模式一致
- 文档/报告统一使用 **云文档 (doc)**
- 通知通道统一使用 **消息 (msg)**

## 三、10 个 Skill 详细规划

### Skill 1: wecomcli-workflow-personal-crm (个人 CRM)

**飞书版依赖：** mail + calendar + contact + base + minutes
**企业微信版依赖：** schedule + contact + msg + smartsheet + meeting
**难度：** ★★★

**核心功能：**
- 从企业微信日程中提取会议参与者，自动建立联系人记录
- 通过消息记录提取联系人互动信息
- 维护联系人档案（姓名、公司、职位、关系热度分）
- 支持自然语言查询联系人
- 超期未联系的重要关系通过消息提醒

**与飞书版差异：**
- ❌ 无邮件扫描 → 改用消息记录 (`msg get_message`) 提取联系人
- ❌ 无妙记 → 改用会议详情 (`meeting get_meeting_info`) 提取互动
- ✅ 日程 (`schedule`) + 通讯录 (`contact`) 完全对应

**数据存储（智能表格，2张子表）：**

子表1: 联系人主表
| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | TEXT | 联系人姓名 |
| 公司 | TEXT | 所在公司 |
| 职位 | TEXT | 职位/角色 |
| 来源 | SINGLE_SELECT | 日程/消息/通讯录/手动 |
| 关系类型 | SINGLE_SELECT | 客户/合作伙伴/同事/朋友/行业人脉/其他 |
| 重要程度 | SINGLE_SELECT | 高/中/低 |
| 热度分 | TEXT | 0-100 (数字以文本存储) |
| 最后联系时间 | TEXT | 日期字符串 |
| 最后联系内容 | TEXT | 互动摘要 |
| 备注 | TEXT | 自由备注 |

子表2: 互动记录
| 字段 | 类型 | 说明 |
|------|------|------|
| 关联联系人 | TEXT | 联系人姓名 |
| 互动时间 | TEXT | 日期字符串 |
| 互动类型 | SINGLE_SELECT | 消息/会议/日程/其他 |
| 互动摘要 | TEXT | 内容简述 |
| 涉及主题 | TEXT | 讨论的话题 |

**关键命令流：**
```bash
# 从日程提取联系人
wecom-cli schedule get_schedule_list_by_range '{"start_time": "...", "end_time": "..."}'
wecom-cli schedule get_schedule_detail '{"schedule_id_list": ["..."]}'

# 从消息记录提取互动
wecom-cli msg get_msg_chat_list '{"begin_time": "...", "end_time": "..."}'
wecom-cli msg get_message '{"chat_type": 1, "chatid": "...", "begin_time": "...", "end_time": "..."}'

# 通讯录查询
wecom-cli contact get_userlist '{}'

# 写入智能表格
wecom-cli doc smartsheet_add_records '{"docid": "...", "sheet_id": "...", "records": [...]}'

# 发送提醒
wecom-cli msg send_message '{"chat_type": 1, "chatid": "...", "msgtype": "text", "text": {"content": "..."}}'
```

**触发词：** "联系人管理" "CRM" "谁联系过我" "多久没联系" "联系人热度" "关系管理"

### Skill 2: wecomcli-workflow-meeting-todo (会议待办追踪)

**飞书版依赖：** minutes + vc + task + im
**企业微信版依赖：** meeting + todo + msg + smartsheet
**难度：** ★★☆

**核心功能：**
- 从会议详情中提取待办事项（我方 + 对方）
- 通过消息确认后自动创建待办
- 定期检查完成情况
- 超 14 天未完成的待办自动归档

**与飞书版差异：**
- ❌ 无妙记自动提取 → 改为用户手动输入待办，或从会议详情描述中 AI 提取
- ❌ 无卡片交互 → 用文本消息确认
- ✅ 待办 (`todo`) 和会议 (`meeting`) 完全对应

**数据存储（智能表格，1张子表）：**

子表: 会议待办追踪
| 字段 | 类型 | 说明 |
|------|------|------|
| 会议主题 | TEXT | 来源会议标题 |
| 会议日期 | TEXT | 会议时间 |
| 待办内容 | TEXT | 提取的待办描述 |
| 待办类型 | SINGLE_SELECT | 我方待办 / 对方待办 |
| 负责人 | TEXT | 谁来做 |
| 建议截止日 | TEXT | 日期字符串 |
| 确认状态 | SINGLE_SELECT | 待确认/已确认/已跳过/已完成/已归档 |
| 关联待办ID | TEXT | 企业微信 todo_id |
| 会议ID | TEXT | meeting_id |

**关键命令流：**
```bash
# 获取会议列表
wecom-cli meeting list_user_meetings '{"begin_datetime": "...", "end_datetime": "..."}'

# 获取会议详情（AI 从描述中提取待办）
wecom-cli meeting get_meeting_info '{"meetingid": "..."}'

# 创建待办
wecom-cli todo create_todo '{"content": "...", "remind_time": "..."}'

# 检查待办状态
wecom-cli todo get_todo_list '{}'
wecom-cli todo get_todo_detail '{"todo_id_list": ["..."]}'

# 发送确认消息
wecom-cli msg send_message '{"chat_type": 1, "chatid": "...", "msgtype": "text", "text": {"content": "..."}}'
```

**触发词：** "会议待办" "会后的待办" "追踪待办" "提取会议行动项" "整理会议纪要待办"

**数据存储（智能表格，1张子表 + 文档）：**

子表: 知识库索引

**关键命令流：**

wecom-cli doc smartsheet_add_records '{"docid": "...", "sheet_id": "...", "records": [...]}'

## 七、共享设计原则

1. **统一 SKILL.md 格式** — 遵循现有 `wecomcli-crm` 的 frontmatter + 前置条件 + 工作流 + 权限表结构
2. **数据全部沉淀到智能表格** — 每个需要持久化的 Skill 都有自己的 smartsheet 表，首次使用时自动创建
3. **通知统一走企业微信消息** — 所有主动推送通过 `wecom-cli msg send_message` 发送
4. **外部数据通过 WebSearch** — 社交媒体、股票等外部数据先用 Claude 的 WebSearch 获取
5. **文档归档到企业微信云文档** — 报告、分析结果等长期内容用 `wecom-cli doc` 存储
6. **首次使用自动初始化** — 每个 Skill 首次触发时，自动创建所需的智能表格和子表
7. **配置保存到 memory** — 表 URL、docid、sheet_id 等配置保存到 memory 文件，避免重复询问
8. **字段写入格式** — 严格遵循现有 CRM 的字段类型格式规则（TEXT 用 `[{"type":"text","text":"..."}]`，SINGLE_SELECT 用 `[{"text":"..."}]`）

## 八、权限配置

需要在 `.claude/settings.local.json` 中追加权限：

```json
{
  "permissions": {
    "allow": [
      "Bash(wecom-cli:*)",
      "Bash(npm list:*)",
      "Bash(npm root:*)"
    ]
  }
}
```

现有配置已包含 `wecom-cli:*`，所有新增 Skill 的命令调用无需额外权限配置。
