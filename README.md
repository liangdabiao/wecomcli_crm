# WeCom Workflow Skills

基于 [wecom-cli](https://github.com/WecomTeam/wecom-cli) + [Claude Code / OpenClaw Skill] 的企业微信智能工作流系统。

通过自然语言驱动企业微信智能表格、文档、日程、待办、消息等能力，实现 10 大自动化工作流。

> 将 Claude Agent/OpenClaw 10 大实战用例完整迁移到企业微信生态，适合 Claude Agent Skill 和 OpenClaw，用企业微信各模块实现所有工作流功能。
> 特别感谢linuxdo的佬友支持：https://linux.do/t/topic/1868703

## 快速开始

### 1. 安装依赖

```bash
npm install -g @wecom/cli
npx skills add WeComTeam/wecom-cli -y -g
```

### 2. 配置凭证

```bash
wecom-cli init    # 交互式输入企业微信机器人 Bot ID 和 Secret
```

### 3. 使用 Skill

在 Claude Code/ OpenClaw  中用自然语言说即可，AI 会自动识别意图并触发对应 Skill。首次使用时，AI 会自动创建智能表格并保存配置。

注意： 如果需要抓取社交网络数据，那么我推荐下载如下skills:
https://github.com/apify/agent-skills

https://github.com/liangdabiao/tikhub_api_skill

## 通用使用说明

### 首次使用

所有 Skill 在首次触发时会自动完成初始化：

1. **创建智能表格** — 自动在企业微信文档中创建对应表格
2. **创建子表和字段** — 按预设 schema 建立数据结构
3. **保存配置到 memory** — 将 docid、url、sheet_id 等保存到本地，后续无需重复配置

### 数据存储

- 所有数据统一存储在**企业微信智能表格**（doc_type=10）
- 分析报告等长文本保存为**企业微信文档**（doc_type=3）
- 配置信息持久化到 Claude Code memory，跨会话保留

### 字段类型

| 字段类型 | 写入格式 | 示例 |
|----------|---------|------|
| TEXT | `[{"type":"text","text":"值"}]` | 食物列表、日期、备注 |
| SINGLE_SELECT | `[{"text":"选项文本"}]` | 餐次、状态、严重程度 |
| PHONE_NUMBER | 直接传字符串 `"138xxxx"` | 手机号 |
| USER | `[{"userid":"用户ID"}]` | 参与人 |

> 选项在首次写入记录时自动创建，无需预配置。

## 各 Skill 详细使用指南

### 1. 健康日记 (health-diary)

**一句话**：记录饮食和身体状态，AI 发现食物与健康的隐藏关联。

**触发说法**：记录饮食、健康日记、吃了什么、身体不舒服、健康追踪、午餐/晚餐吃了

#### 模式 1：文字记录饮食

```
你：午餐吃了红烧牛肉面和一杯冰美式
AI：确认一下：
    - 餐次：午餐
    - 食物：红烧牛肉面、冰美式
    - 份量：一大碗、大杯
    - 热量估算：约 650 大卡
    记录吗？
你：确认
AI：（自动写入智能表格）
```

#### 模式 2：照片识别记录

```
你：（通过企业微信发送食物照片）
AI：识别到：一碗拉面、一份煎饺、一杯橙汁
    份量看起来是常规大小，记录吗？
你：对，拉面是大碗
AI：（更新份量后写入）
```

#### 模式 3：记录身体状态

```
你：下午胃有点不舒服
AI：（自动查询最近 6 小时饮食记录，检查关联）
    已记录胃部不适。注意：中午吃的红烧牛肉面
    与胃部不适的关联强度为"高"（已出现 4 次），
    建议减少辛辣食物摄入。
```

#### 模式 4：关联分析

```
你：分析一下最近的饮食和身体状况
AI：基于最近 30 天的记录，发现以下关联：
    1. 冰美式 → 胃部不适（出现 4 次，强度：高，典型延迟 2 小时）
    2. 辛辣食物 → 消化不良（出现 2 次，强度：中）
```

#### 模式 5：生成健康周报

```
你：生成本周健康周报
AI：（汇总本周数据，生成报告文档）
    周报已生成，包含：
    - 饮食概览（记录 6 天，共 18 餐）
    - 身体状态（2 次不适，主要症状：胃部不适）
    - 关联发现（3 条新关联）
    - 改进建议
```

#### 数据表结构

| 子表 | 关键字段 |
|------|---------|
| 饮食记录 | 日期、餐次、食物列表、份量说明、热量估算、备注 |
| 身体状态 | 记录时间、状态描述、严重程度、持续时长、备注 |
| 关联分析 | 食物、状态、出现次数、典型时间差、关联强度 |

#### 关联强度规则

| 出现次数 | 关联强度 |
|---------|---------|
| 1 次 | 低 |
| 2-3 次 | 中 |
| 4 次以上 | 高 |

### 4. 个人 CRM (personal-crm)

**一句话**：管理所有人脉关系，自动计算热度分，提醒你维护重要联系。

**触发说法**：联系人管理、个人CRM、谁联系过我、多久没联系、联系人热度、关系管理

#### 模式 1：自动扫描提取

```
你：扫描一下最近一周的联系人
AI：（从日程和消息中提取联系人）
    发现 8 位联系人：
    | 姓名 | 来源 | 互动类型 | 时间 |
    |------|------|---------|------|
    | 张三 | 日程 | 会议参会 | 4/2 |
    | 李四 | 消息 | 聊天 | 4/1 |
    | 王五 | 日程 | 会议参会 | 3/30 |
    ...
    其中 3 位为新联系人，是否添加到 CRM？
```

#### 模式 2：自然语言查询

```
你：上个月谁联系过我？
AI：上月互动记录：
    - 张三（XX科技）：4 次互动，最近 3/28
    - 李四：2 次互动，最近 3/25
    - 王五：1 次互动，3/15

你：华南区有哪些客户？
你：给我推荐 3 个最近没联系的重要联系人
你：谁的公司是做 AI 的？
```

#### 模式 3：手动添加/更新

```
你：添加联系人：赵六，某某科技 CEO
AI：已添加。热度分：50（新联系人，基础分）
```

#### 模式 4：超期提醒

```
你：哪些重要联系人超过 30 天没联系了？
AI：以下联系人超过 30 天未互动：
    | 姓名 | 公司 | 职位 | 最后联系 | 热度分 | 状态 |
    |------|------|------|---------|--------|------|
    | 王五 | YY科技 | CTO | 45天前 | 42 ↓ | ⚠️ 下降 |
    | 赵六 | ZZ公司 | VP | 60天前 | 28 ↓↓ | 🔴 危险 |
    建议优先联系王五（热度分下降明显）
```

#### 热度分算法

热度分 = 频率分（40%）+ 近期分（40%）+ 重要度分（20%）

- **频率分**：近 90 天互动次数，每次 +10 分，上限 40
- **近期分**：最后一次互动距今天数，0天=40，7天=35，30天=25，90天=0
- **重要度分**：根据公司规模、职位、互动质量评估，上限 20

#### 数据表结构

| 子表 | 关键字段 |
|------|---------|
| 联系人 | 姓名、公司、职位、电话、邮箱、热度分、最后联系时间、标签 |
| 互动记录 | 联系人、互动类型、互动时间、互动内容摘要 |

> **与 wecomcli-crm 的区别**：个人 CRM 管理所有人脉（同事、合作伙伴、朋友等），wecomcli-crm 专注商业客户管理。

### 10. 多代理协同 (multi-agent-dev)

**一句话**：启动多个 AI 代理并行编程，支持竞赛对比、分工并行、流水线协作。

**触发说法**：并行开发、多代理、竞赛模式、分工模式、流水线协作、代码对比

#### 模式 1：竞赛模式

```
你：用竞赛模式实现一个用户登录模块
AI：（启动 2 个隔离代理，各自独立实现）

    ## 竞赛结果

    | 维度 | 代理1（JWT方案） | 代理2（Session方案） |
    |------|-----------------|-------------------|
    | 认证方式 | JWT + Refresh Token | Session-based |
    | 代码质量 | 8.5 | 7.0 |
    | 完整性 | 含密码强度校验 | 缺少密码强度校验 |
    | 安全性 | 9.0 | 7.5 |

    推荐：代理1（JWT方案）
    原因：更符合现代最佳实践，功能更完整

    是否采用代理1的方案？
```

#### 模式 2：分工模式

```
你：并行做 3 个任务：修 bug #123、写登录页面、优化 SQL 查询
AI：（启动 3 个代理并行执行）

    ## 分工任务完成
    | 任务 | 代理 | 状态 | 耗时 |
    |------|------|------|------|
    | 修 bug #123 | Agent1 | 完成 | 45s |
    | 写登录页面 | Agent2 | 完成 | 120s |
    | 优化 SQL 查询 | Agent3 | 完成 | 60s |

    总耗时 120s（串行需 225s，加速 1.9x）
```

#### 模式 3：流水线模式

```
你：用流水线模式重构用户模块
AI：（按顺序执行：实现 → 审查 → 测试）

    阶段1：代理1 完成代码重构
    阶段2：代理2 审查发现 3 个问题：
      1. 错误处理不够全面
      2. 缺少输入校验
      3. 命名不一致
    阶段3：代理3 补充了 12 个单元测试，覆盖率 92%

    最终结果已合并，报告已保存
```

#### 三种模式对比

| 模式 | 适用场景 | 执行方式 | 优势 |
|------|---------|---------|------|
| 竞赛 | 需要最优方案、架构选择 | 同一任务，各自完成，AI对比 | 获得最佳方案 |
| 分工 | 多个独立任务、批量开发 | 不同任务，并行完成 | 时间加速 Nx |
| 流水线 | 复杂功能、质量保障 | 代码→审查→测试，顺序协作 | 质量有保障 |

| 竞赛记录 | 任务描述、代理1/2方案、评分、推荐方案、对比分析 |

## 常见问题

### Q：首次使用需要做什么？
A：不需要手动配置。在 Claude Code 中用自然语言描述你的需求，AI 会自动识别对应的 Skill，首次使用时自动创建智能表格并保存配置。

### Q：数据存在哪里？
A：所有数据存储在企业微信智能表格中，分析报告等长文本保存为企业微信文档。配置信息存储在本地 Claude Code memory 中。

### Q：多个 Skill 之间怎么联动？
A：Skill 之间通过读取彼此的智能表格数据实现联动。例如商业顾问团会读取个人 CRM、社交追踪、内容管道、股票研究等 Skill 的数据。只有已初始化的 Skill 会被读取，未初始化的自动跳过。

### Q：Skill 可以定时自动运行吗？
A：可以。通过 Claude Code 的 Cron 功能设置定时任务，建议的自动化计划：

| 任务 | 建议时间 | Cron |
|------|---------|------|
| 晨间简报 | 工作日 8:00 | `0 8 * * 1-5` |
| 持仓快照 | 工作日 18:00 | `0 18 * * 1-5` |
| 关联分析更新 | 每日 22:00 | `0 22 * * *` |
| 健康周报 | 每周日 20:00 | `0 20 * * 0` |
| 商业顾问团分析 | 工作日 22:00 | `0 22 * * 1-5` |

### Q：企业微信没有邮件/知识库功能怎么办？
A：系统已做适配：
- 邮件 → 用消息记录替代
- 知识库 → 智能表格做索引 + 文档做内容存储
- 飞书多维表 → 企业微信智能表格
- 飞书文档 → 企业微信文档

### Q：API 调用报错怎么办？
A：常见错误码：
- **301085** — docid 无效，改用 url 参数
- **2022004** — 字段未找到，先执行 `smartsheet_add_fields`
- **2022003** — 记录不存在，先执行 `smartsheet_add_records`
